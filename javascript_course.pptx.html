TOPIC 04

JavaScript Operators

Symbols that perform operations on values

Arithmetic Operators

let a = 10, b = 3;

a + b // 13 (Addition)

a - b // 7 (Subtraction)

a * b // 30 (Multiplication)

a / b // 3.33 (Division)

a % b // 1 (Modulus)

a ** b // 1000 (Exponent)

Assignment Operators

let x = 10;

x += 5; // x = x + 5 (15)

x -= 3; // x = x - 3 (12)

x *= 2; // x = x * 2 (24)

x /= 4; // x = x / 4 (6)

x %= 4; // x = x % 4 (2)

Comparison Operators

5 == "5" // true (loose)

5 === "5" // false (strict)

5 != "5" // false

5 !== "5" // true

10 > 5 // true

10 <= 10 // true

Ternary Operator

// condition ? value_if_true : value_if_false

let age = 18;

let status = age >= 18 ? "adult" : "minor";

console.log(status); // "adult"

Unary Operators

let x = 5;

x++; // Post-increment (returns 5, x=6)

++x; // Pre-increment (returns 7, x=7)

-x; // Negation (-7)

+"5"; // Convert to number (5)

!true; // Logical NOT (false)

TOPIC 06

Loops in JavaScript

Repeat code execution efficiently

for Loop

When you know the iteration count.

// for (init; condition; increment)

for (let i = 0; i < 5; i++) {

console.log(i); // 0,1,2,3,4

}

while Loop

When iterations are unknown.

let count = 0;

while (count < 5) {

console.log(count);

count++;

}

for-in Loop

Iterate over object keys.

let obj = {a: 1, b: 2};

for (let key in obj) {

console.log(key, obj[key]);

}

// Output: "a" 1, "b" 2

for-of Loop

Iterate over iterable values.

let arr = ["a", "b", "c"];

for (let value of arr) {

console.log(value);

}

// Output: "a", "b", "c"

// Works with: Arrays, Strings, Maps, Sets

Key Difference: for-in iterates over keys (indices/properties), for-of iterates over values. Use for-of for arrays, for-in for objects.

TOPIC 08

Arrays in JavaScript

Ordered collections of data

Basic Array Methods

let arr = ["a", "b", "c"];

arr.push("d"); // Add to end

arr.pop(); // Remove from end

arr.unshift("z"); // Add to start

arr.shift(); // Remove from start

arr.indexOf("b"); // Find index (1)

arr.includes("b"); // Check (true)

arr.slice(1, 3); // Extract ["b","c"]

arr.splice(1, 1); // Remove at index

sort() Method

let nums = [3, 1, 4, 1, 5];

nums.sort(); // [1,1,3,4,5] (as strings!)

// Correct numeric sort

nums.sort((a, b) => a - b);

// Descending: b - a

Iteration Methods

let nums = [1, 2, 3];

// forEach - execute for each

nums.forEach(n => console.log(n));

// map - transform each

let doubled = nums.map(n => n * 2);

// filter - keep matching

let evens = nums.filter(n => n % 2 === 0);

// find - first match

let first = nums.find(n => n > 1); // 2

// every/some - test all/any

nums.every(n => n > 0); // true

Array Tip: Methods like map(), filter(), reduce() don't mutate the original array — they return new arrays!

names.sort((a, b) =>

TOPIC 14

Storing Data in the Browser

Client-side storage options

Cookies

Small data sent with HTTP requests. Max ~4KB.

// Set cookie

document.cookie = "username=John; expires=...; path=/";

// Read all cookies

console.log(document.cookie);

// Delete cookie

document.cookie = "username=; expires=Thu, 01 Jan 1970";

LocalStorage

Persistent storage (~5-10MB). No expiration.

// Store data

localStorage.setItem("username", "John");

// Retrieve data

let user = localStorage.getItem("username");

// Remove item

localStorage.removeItem("username");

// Clear all

localStorage.clear();

// Store objects

let obj = {name: "John", age: 30};

localStorage.setItem("user", JSON.stringify(obj));

let data = JSON.parse(localStorage.getItem("user"));

sessionStorage

Temporary storage. Cleared when tab closes.

// Same API as localStorage

sessionStorage.setItem("temp", "value");

sessionStorage.getItem("temp");

sessionStorage.removeItem("temp");

Comparison

Cookies

Small, sent to server, can expire

LocalStorage

Large, persistent, client-only

SessionStorage

Large, tab-specific, temporary

Note: All storage is limited to the current domain (same-origin policy). Data is not shared across different websites.

Default, Rest & Spread

function greet(name = "Guest") {

let [first, ...rest] = [1, 2, 3, 4];

// Object destructuring

let {name, age} = {name: "John", age: 30};

// With renaming

let {name: userName} = {name: "John"};

// Default values

let {role = "user"} = {name: "John"};

// Nested destructuring

let {user: {name}} = {user: {name: "John"}};

TOPIC 16

Module

Organizing code with ES6 modules

Arrow Functions

Concise syntax for function expressions.

// Traditional function

const add = function(a, b) {

return a + b;

};

// Arrow function

const add = (a, b) => a + b;

// Single parameter (no parentheses)

const square = x => x * x;

// Multiple statements

const greet = name => {

console.log("Hello!");

return `Hi, ${name}`;

};

Exporting Modules

// math.js - Named exports

export const PI = 3.14159;

export function add(a, b) {

return a + b;

}

export default function multiply(a, b) {

return a * b;

}

Importing Modules

// Import default export

import multiply from './math.js';

// Import named exports

import { PI, add } from './math.js';

// Import with alias

import { add as sum } from './math.js';

// Import all exports

import * as math from './math.js';

math.add(2, 3); // 5

// HTML setup

<script type="module" src="app.js"></script>

Module Benefits: Code organization, reusability, encapsulation, and dependency management. Use type="module" in script tag.

TOPIC 17

Asynchronous JavaScript

Handling operations that take time

Promises

Object representing eventual completion.

let promise = new Promise((resolve, reject) => {

let success = true;

if (success) {

resolve("Done!");

} else {

reject("Error!");

}

});

// Consuming promises

promise

.then(result => console.log(result))

.catch(error => console.error(error))

.finally(() => console.log("Complete"));

Promise Chaining

fetch('/api/user')

.then(response => response.json())

.then(data => console.log(data.name))

.catch(error => console.error("Failed:", error));

// Promise.all - wait for all

Promise.all([

fetch('/api/users'),

fetch('/api/posts')

]).then(([users, posts]) => {

// Both completed

});

Async/Await

Syntactic sugar for promises. Cleaner code.

// Async function

async function getUser() {

try {

let response = await fetch('/api/user');

let data = await response.json();

console.log(data);

} catch (error) {

console.error("Error:", error);

}

}

// Arrow async function

const getData = async () => {

let result = await fetchData();

return result;

};

// Parallel execution with await

const [users, posts] = await Promise.all([

fetchUsers(),

fetchPosts()

]);

Best Practice: Use async/await for cleaner asynchronous code. It makes async code look and behave more like synchronous code.

TOPIC 18

Error Handling

Gracefully managing errors

try...catch Statement

try {

// Code that might throw error

let result = riskyOperation();

console.log(result);

} catch (error) {

// Handle the error

console.error("An error occurred:", error.message);

} finally {

// Always executes

console.log("Cleanup code here");

}

Throwing Exceptions

function divide(a, b) {

if (b === 0) {

throw new Error("Cannot divide by zero!");

}

return a / b;

}

try {

console.log(divide(10, 0));

} catch (e) {

console.error(e.name); // "Error"

console.error(e.message); // "Cannot divide..."

}

Error Types

Error

Generic error

SyntaxError

Invalid syntax

ReferenceError

Undefined variable

TypeError

Wrong type operation

RangeError

Value out of range

Best Practice: Always handle errors gracefully. Use try-catch for risky operations and provide meaningful error messages to users.
